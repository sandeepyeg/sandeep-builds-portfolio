# Deployment

## Azure Static Web Apps Setup

### 1. Create Azure Static Web App

In the Azure Portal:

1. Create a Static Web App resource
2. Connect to your GitHub repository
3. Configure:

| Setting         | Value                         |
| --------------- | ----------------------------- |
| App location    | `/`                           |
| API location    | _(leave empty)_               |
| Output location | `dist/sandeep-builds/browser` |
| Branch          | `main`                        |

### 2. GitHub Connection

Azure will automatically create a deployment workflow in `.github/workflows/`. The token secret name will be generated by Azure — preserve the generated name.

### 3. Build Output

Run `npm run build:production` to verify the output directory:

```
dist/sandeep-builds/browser/
  index.html
  404.html
  work/enterprise-payments/index.html
  ...
```

### 4. Custom Domain

1. In Azure Portal → Static Web App → Custom domains
2. Add `sandeepbuilds.com` and `www.sandeepbuilds.com`
3. Azure will display DNS validation values
4. Configure DNS records with your domain registrar:
   - CNAME record for `www` pointing to the Azure-provided URL
   - TXT record for domain validation
   - For apex domain, use the Azure-provided A/ALIAS records
5. Wait for DNS propagation and HTTPS certificate provisioning

### 5. Canonical Redirect

Choose one canonical domain (recommended: `https://sandeepbuilds.com`). Configure the `www` subdomain to redirect to the apex domain via Azure's domain settings.

### 6. Environment / Secrets

No environment secrets are needed for the static build. If a contact form is added later with Azure Functions, add the function app URL as a secret.

### 7. Rollback

Revert the commit on `main` and push. Azure will redeploy the previous version automatically.

### 8. Common Issues

- **404 on subroutes**: Verify `staticwebapp.config.json` has `navigationFallback.rewrite: "/index.html"` and the excluded routes match actual static pages
- **Build fails on output location**: Run `npm run build:production` locally and check the exact `outputPath` in `angular.json`
- **DNS not resolving**: Propagation can take up to 48 hours; verify with `dig sandeepbuilds.com`
- **HTTPS not provisioning**: Wait up to 24 hours after DNS validation; ensure no conflicting CAA records
